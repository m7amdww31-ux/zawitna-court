# ⚖️ بوت "الزاوية تحاكم" — نسخة Slash Commands

محكمة هزلية لسيرفر **زاويتنا**، القاضي فيها هو الذكاء الصناعي (كلود).
الأوامر تطلع تلقائياً بقائمة منسّقة أول ما تكتبين `/` في ديسكورد.

---

## 🧩 الأوامر
| الأمر | الوظيفة |
|------|---------|
| `/محاكمة` (المتهم + التهمة) | يرفع قضية جديدة |
| `/دفاع` (نص) | المتهم يدافع عن نفسه |
| `/حكم` | القاضي يصدر الحكم النهائي |
| `/تنازل` | صاحب القضية يغلق المحاكمة |
| `/مساعدة` | يعرض الأوامر |

---

## 📦 المتغيّرات (Variables) في Railway

| Name | Value | إلزامي؟ |
|------|-------|---------|
| `DISCORD_TOKEN` | توكن البوت من Developer Portal | ✅ نعم |
| `ANTHROPIC_API_KEY` | مفتاح كلود `sk-ant-...` | ✅ نعم |
| `MISE_PYTHON_GITHUB_ATTESTATIONS` | `false` | ✅ نعم |
| `GUILD_ID` | آيدي سيرفر زاويتنا (رقم) | ⭐ اختياري بس مُفضّل |
| `DEFENSE_TIMEOUT` | ثواني انتظار الدفاع قبل الحكم الغيابي (مثل `120`) | اختياري |

> **الحكم التلقائي:** أول ما المتهم يكتب `/دفاع`، القاضي يصدر حكمه **فوراً** — بدون ما أحد يكتب `/حكم`.
>
> **`DEFENSE_TIMEOUT`:** لو حطّيتيه برقم (مثل `120`)، وما دافع المتهم خلال هالمدة، القاضي يحكم عليه **غيابياً تلقائياً**. لو خليتيه فاضي أو `0` → ما فيه حكم غيابي تلقائي (تستخدمين `/حكم` يدوياً وقت ما تبين).

> **ليش `GUILD_ID` مُفضّل؟** لو حطّيتيه، الأوامر تظهر **فوراً** في سيرفرك.
> لو ما حطّيتيه، البوت يسجّلها بشكل عام وقد تاخذ حتى ساعة عشان تبين.

### كيف تجيبين `GUILD_ID`؟
1. في ديسكورد: **User Settings** → **Advanced** → فعّلي **Developer Mode**.
2. ارجعي للسيرفر، اضغطي كليك يمين على أيقونة سيرفر **زاويتنا** → **Copy Server ID**.
3. هذا الرقم هو قيمة `GUILD_ID`.

---

## 1️⃣ تجهيز بوت ديسكورد

1. افتحي **Discord Developer Portal**: https://discord.com/developers/applications
2. **New Application** → اكتبي اسم (`محكمة زاويتنا`) → **Create**.
3. من الجانب اضغطي **Bot** → تحت **Token** اضغطي **Reset Token** → **Yes, do it!** → انسخي التوكن (هذا `DISCORD_TOKEN`).

> 🎉 في هالنسخة **ما تحتاجين تفعّلين أي Intents** — السلاش كوماندز تشتغل بدونها.

### دعوة البوت (بصلاحية السلاش كوماندز)
4. من الجانب: **OAuth2** → **URL Generator**.
5. تحت **SCOPES** علّمي ✅ `bot` **و** ✅ `applications.commands` (الثنتين مهمين!).
6. تحت **BOT PERMISSIONS** علّمي ✅ `Send Messages`.
7. انسخي الرابط تحت، افتحيه، اختاري سيرفر **زاويتنا** → **Authorize**.

> ⚠️ مهم: لو كنتِ داعية البوت قبل بصلاحية `bot` بس، **افتحي الرابط الجديد مرة ثانية** عشان تنضاف صلاحية `applications.commands`. ما يتكرّر البوت ولا ينطرد — بس تنضاف الصلاحية.

---

## 2️⃣ مفتاح كلود (Anthropic)
1. https://console.anthropic.com → **API Keys** → **Create Key** → انسخيه (`ANTHROPIC_API_KEY`).
2. تأكدي من وجود رصيد في **Billing** وإلا أمر `/حكم` ما يشتغل.

---

## 3️⃣ رفع الملفات على GitHub
1. ريبو جديد (`zawiya-court`) → **Add file** → **Upload files**.
2. ارفعي `bot.py` و `requirements.txt` → **Commit changes**.

---

## 4️⃣ النشر على Railway
1. https://railway.app → **New Project** → **Deploy from GitHub repo** → اختاري الريبو.
2. اضغطي على الخدمة → **Variables** → **New Variable** وأضيفي المتغيّرات من الجدول فوق.
3. لو ما اشتغل تلقائياً: **Settings** → **Deploy** → **Custom Start Command** → `python bot.py`.
4. من **Deploy Logs** لو طلع:
   ```
   ⚖️ المحكمة جاهزة — دخل القاضي باسم: ...
   ```
   معناها شغّال ✅، وأوامر الـ `/` بتكون ظهرت في السيرفر (فوراً لو حطّيتي GUILD_ID).

## 🧪 جرّبيه
اكتبي في أي قناة:
```
/محاكمة  المتهم:@أحد  التهمة: سرق آخر قطعة بيتزا
```
المتهم يرد:
```
/دفاع  نص: والله ما كنت أنا، كنت نايم
```
وبمجرّد ما يرسل الدفاع، القاضي يصدر حكمه الدرامي **فوراً تلقائياً** 😈⚖️
(ولو تبين تستعجلين الحكم بدون انتظار الدفاع، اكتبي `/حكم`).

## ⚙️ ملاحظات
- ما يحتاج قاعدة بيانات ولا Volume — كل قضية مؤقتة وتنتهي بالحكم.
- للقاضي الأسرع/الأرخص: في `bot.py` غيّري `MODEL = "claude-sonnet-4-6"` إلى `"claude-haiku-4-5-20251001"`.
- قضية وحدة مفتوحة بكل قناة بنفس الوقت (تقدرين تشغّلين قضايا متوازية بقنوات مختلفة).
